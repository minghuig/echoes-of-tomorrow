# DECISIONS

Architectural choices, dated, with one-line rationale. Newest at the bottom.

- **2026-07-04 — Godot 4.x standard build.** Standard (non-.NET) build keeps web export working and avoids a C# build step.
- **2026-07-04 — Typed GDScript over C#.** C# blocks web export and adds build-step friction for agents; typed GDScript keeps iteration instant.
- **2026-07-04 — 2D top-down.** One control paradigm across all modes per the scope covenants; no camera/perspective complexity.
- **2026-07-04 — 60Hz fixed tick.** Sim advances only in exact 1/60s steps; frame rate never touches game logic.
- **2026-07-04 — Deterministic sim core with command-based input.** A run is `(seed, command_log)`; ghosts, rewind, and replays are re-runs of the sim with different command sources.
- **2026-07-04 — Codename ASSET-7.** Working title per VISION.md.
- **2026-07-04 — Engine version pinned to 4.6.3-stable.** Local binary and CI image (`barichello/godot-ci:4.6.3`) must match so exports and tests behave identically.
- **2026-07-04 — Cross-file sim references use `preload()` consts, not bare `class_name` lookups.** Global class resolution needs the editor's `.godot/` cache; preloads keep headless/CI runs working from a cold checkout.
- **2026-07-05 — Renamed to Echoes of Tomorrow.** ASSET-7 retired as the project title; "ASSET 7" survives only as the player AI's in-fiction designation (the credits stinger in VISION.md).
- **2026-07-05 — Per-run data fragments live in SimState.** The sim is authoritative for what a run earns (1 per destroyed block), and the field is in `serialize()` so earnings sit inside the determinism contract.
- **2026-07-05 — Meta/persistence state lives in the view layer (`view/run_meta.gd` → `user://save.json`), never in the sim.** Lifetime fragments and run count are meta-game knowledge across runs; keeping them (and all disk I/O) outside `sim/` preserves "a run = (seed, command log)" and keeps replays/ghosts from depending on save files. Nothing persisted feeds back into sim behavior yet.
- **2026-07-05 — Seed selection is meta-layer policy: `BASE_SEED + lifetime run index`.** The view picks each run's seed before constructing the sim; the sim never chooses or mutates its own seed, so any run replays from its recorded `(seed, command_log)`.
- **2026-07-05 — M2 win condition is a meta-layer fragment threshold, not sim state.** Lifetime fragments ≥ `meta.win_fragment_target` (tuning data) rolls credits in the view; the sim keeps only the per-run clear, so determinism and the M3 replay substrate are untouched.
- **2026-07-05 — Ghosts are parallel SimCores re-running a recorded `(seed, command_log)`.** The view steps the ghost one recorded command per live tick and renders it translucent; ghost state is never merged into or read by the live sim. First working proof of "everything is a re-run with a different command source".
- **2026-07-05 — Juice dilates view time, never sim time.** Hit-stop skips the entire view tick (no command recorded, no sim step, ghost included), so command logs stay 1:1 with sim ticks; shake jitter uses a view-local RNG, never the sim's.
- **2026-07-05 — SFX are synthesized PCM at startup (`view/sfx.gd`), not asset files.** Square-wave blips keep the repo asset-free and the web export tiny; audio is pure view flavor the sim never sees.
- **2026-07-05 — The beach has no clear condition; waves escalate forever.** Act 1's "unwinnable beach" is mechanical truth: authored waves then linear escalation, all data in `content/waves.json`. Runs end in death (or manual reset), and death banks fragments — losing is the progression loop (Vampire Survivors pressure curve, Hades death-as-return).
- **2026-07-05 — Wave spawn positions are the sim RNG's first consumer.** Spawn x-jitter draws from the seeded RNG at wave-schedule time only, so runs stay reproducible from `(seed, command_log)` and ghost replays get identical assaults.
- **2026-07-05 — Dodge grants i-frames (`dodge_iframe_ticks` tuning).** The dash is the survival verb against an unwinnable screen (Hades); invulnerability lives in sim state so replays honor it exactly.
- **2026-07-05 — A run is `(seed, loadout, command_log)`.** The sentience tree resolves to a flat stat-modifier dict the view passes into `SimCore.setup()`; the sim never reads save files. RunRecords carry their loadout so ghosts replay under the exact stats they were played with.
- **2026-07-05 — Sentience-tree tiers are absolute, not stacking (Hades Mirror of Night model).** Each owned tier fully defines its branch's effect in `content/sentience_tree.json`; resolution is "take the owned tier's effects", which keeps balance authorable in data and loadout resolution trivial.
- **2026-07-05 — Intel is knowledge-gated authored lore (Outer Wilds ship-log model), unlocked by the lifetime combat record.** Conditions (deaths / kills / wave / fragments / upgrades) live with the entries in `content/intel.json`; `RunMeta.evaluate_intel()` is the single evaluator. Pure view/meta + content — the sim knows nothing about lore.
- **2026-07-05 — Run results bank once, at the moment of death (or manual reset), not at redeploy.** `_bank_run_results()` is idempotent per run; banking early lets the death panel announce fresh decrypts while the corpse is still on screen.
- **2026-07-05 — PR previews deploy to `gh-pages` (`pr/<n>/` per PR, `latest/` for main), not itch.io channels.** Free, secretless, and works the moment Pages is enabled; the web export keeps `thread_support=false` specifically so plain Pages hosting needs no cross-origin-isolation headers.
- **2026-07-05 — Touch controls are view-layer and gated to web + touchscreen (`OS.has_feature("web") and DisplayServer.is_touchscreen_available()`).** They fill the same Command fields as keyboard/mouse — the sim cannot tell input sources apart, so replays and ghosts are untouched. Touch replaces pointer input when active because browsers emulate a mouse from touches, which would fight the virtual stick.
- **2026-07-06 — The view is a holographic training construct rendered by four child nodes (Background / World / Effects / Overlay).** Ported from the M0 art pass (PR #10) and extended to the full M7 entity set: `main.gd` draws nothing, it orchestrates the run and delegates rendering. Background (holo grid/frame/motes + sea band) and Effects (additive particles) sit at fixed z-index; World shakes via node position so the Overlay HUD (on a `CanvasLayer`) stays steady. Training sims aren't photoreal (VISION.md), so glow is faked with layered translucent draws + an additive-blend node because the `gl_compatibility` web renderer has no 2D HDR glow.
- **2026-07-06 — All juice is derived by diffing successive SimStates in the view; the sim gains no events.** Muzzle flashes, impacts, recoil, block/enemy shatter, dodge bursts, afterimages, and projectile tracer trails are computed from `(pre, post)` snapshots each view tick — so any command source (live, ghost, replay) reproduces identical presentation, and the determinism contract is untouched. `sim_core.gd` only made two tuning fields public (`dodge_cooldown_ticks`, `block_max_hp`) for cosmetic indicators, following the existing radii precedent.
- **2026-07-06 — Gradual identity reveal.** The player starts inside a human-time-loop fiction; "it's a sim" and "you're the AI" are separate staged reveals (L2 and L3), each delivered by a mechanic unlock that breaks the old self-story — no UI/dialogue may say AI, sim, process, or training before the L3 flag.
- **2026-07-06 — Player-facing text is data, and linted.** All display strings live in `content/strings.json` (sections prefixed `post_l3` are reveal-gated and exempt); `tests/test_reveal_discipline.gd` fails CI if pre-reveal strings, string literals in `view/` scripts, or project metadata use banned vocabulary, so the reveal discipline is enforced mechanically, not by memory.
- **2026-07-06 — Pause is a fourth view `Mode`, not a new sim state.** `Mode.PAUSED` freezes `main.gd`'s tick loop (no command built, no `SimCore.step()`, matching the existing hit-stop precedent) instead of teaching the sim to pause; it's entered/exited by a new `pause` action (Escape + gamepad Start) checked ahead of `reset` so Start resolves to pause, not `_end_run()`, while a run is live. `reset` keeps its prior meaning everywhere else (death panel, Between, credits) since pause is only intercepted in `Mode.PLAYING` while the player is alive.
- **2026-07-06 — Overlay hints follow the last-used input device, tracked from raw events, not polling.** `main.gd` flips `_using_gamepad` off key/mouse-button events and on joypad button/motion events past a deadzone (drift-resistant), and pushes it to `Overlay` each tick the same way `touch` already is. Button-hint strings pick touch → gamepad → keyboard in that priority, mirroring the touch-only branch that already existed for the death-panel prompt.
- **2026-07-06 — Window size is a view-only setting, resolved through `canvas_items` stretch, never a sim concern.** `view/display_settings.gd` follows `RunMeta`'s load/save-to-`user://` pattern to persist a chosen preset (or fullscreen); the base viewport stays 1280x720 in `project.godot` and Godot's existing `canvas_items` stretch mode does the scaling, so no sim/gameplay coordinate ever depends on the window size the player picked.
- **2026-07-06 — Save reset lives behind a two-step confirm on the Pause screen, not a separate menu.** `_erase_armed_ticks` in `main.gd` arms on one `buy` press and auto-cancels after 3 seconds (or an explicit `pause` press) unless confirmed by a second `buy` press, which writes a fresh `RunMeta` to `user://save.json` and starts a new run immediately — the only way to reset lifetime progress, since `RunMeta` otherwise only ever grows. **Superseded same day** by the 3-slot save system below.
- **2026-07-06 — Progress lives in 3 independent save-file slots (`user://save_slot_<1|2|3>.json`), not one growing file.** `RunMeta.load_from_disk()`/`save_to_disk()` take an optional path (default preserved) so `main.gd` can point them at whichever slot is active (`_active_slot`, itself persisted to `user://active_slot.json`). The pre-slots `user://save.json` migrates into slot 1 on first boot (`_migrate_legacy_save()`) so nobody's progress is stranded by the scheme change. "Start a new game" is now switching to an empty slot from the Pause screen's slot list — non-destructive, replacing the erase-confirm above, which is why that entry was superseded rather than deleted.
- **2026-07-06 — Pause is reachable from the Between, not just live play; `_mode_before_pause` decides what it resumes to.** The Between has no pause/settings access of its own, and display + save-slot settings apply everywhere, so `Mode.BETWEEN` now also checks the `pause` action. `Overlay` picks which frozen screen to dim (beach vs. Between) from `mode_before_pause`, pushed alongside `mode` each tick.
- **2026-07-06 — Confirmed (not assumed) that window resize/fullscreen work correctly via Godot's own APIs; the "doesn't work on Mac" report traced to the editor's embedded Play window, not the code.** Diagnostic scripts driving `Window.size`/`Window.mode` directly and through the real Pause-screen input path both resized/fullscreened correctly on this machine when run as a standalone binary; `ps aux` on the same machine showed the actual reported Play session running with `--embedded --wid <id>` — a real OS window never exists for it to resize. No code change addresses this; logged in BACKLOG.md as a known editor-configuration limitation.
